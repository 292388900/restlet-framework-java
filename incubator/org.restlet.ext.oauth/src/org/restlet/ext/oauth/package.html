Support for OAuth 2.0 HTTP authentication. This is intended to be used with primarily following use-cases in mind:
  • Create a protected resource
  • Create a authenticating server
  • Create a web client accessing protected resources
It is very simple to create an OAuth server with just a few lines of code. It is also possible to implement a custom back end for data storage and retrieval. The current implementation stores only to memory, so a JVM restart flushes all data.

The package also includes a OpenID Relay Proxy integrated using the OpenID4Java project. It is not mandated to use OpenID as the identification technology, but at the moment it is the only verified solution. The code also provides an OpenId provider that is strictly for basic testing

Example invocation protected resource: 

{@code
public Restlet createInboundRoot(){
  ...
  LocalAuthorizer auth = new LocalAuthorizer(
             "/authorize",
             "http://localhost:8080/OAuth2Provider/validate");
  auth.setNext(ProtectedResource.class);
  router.attach("/me", auth);
  ...
}
}
