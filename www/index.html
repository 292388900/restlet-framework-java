Mission

The goal of this project is to bring the power and simplicity of the REST architectural style into the hands of Java developers.

Features

  • Pure REST API specifying all concepts (Resource, Representation, Data, Connector, Components, etc.)
  • Restlet and Maplet concepts provide a powerful alternative to Servlets
  • Neutral Restlet API with no external dependency
  • Support blocking and non-blocking input/output APIs (IO and NIO)
  • Reference implementation provided: Noelios Restlet Server
  • Protocol independance of the Restlet API. The reference implementation currently supports:
    • Server connector: HTTP
    • Client connectors: JDBC, SMTP, POP3
    • Server-driven content negociation
    • Automatic MIME type, language and character set detection based on file extension

    Requirements

    • Java SE 5.0 (for enumerations and generics)
    • Jetty 5.1.5 to use the server HTTP connector
    • FreeMarker 2.3 to use the FreeMarkerRepresentation (dynamic templates similar to JSP)
    • JavaMail 1.3 to use the client JavaMail connector
